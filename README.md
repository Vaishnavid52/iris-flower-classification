## рҹ§  What this project does

Given four measurements of an iris flower вҖ” sepal length, sepal width, petal length, petal width вҖ” the system predicts which of three species it belongs to (*Setosa*, *Versicolor*, *Virginica*) with a confidence score, and visualizes the result against all 150 flowers in the training set.

This isn't just a notebook. It's a full pipeline:

```
Raw data вҶ’ EDA вҶ’ Model training & tuning вҶ’ REST API вҶ’ Interactive frontend
```

## рҹ”¬ Phase 1 вҖ” Exploratory Data Analysis

Full statistical and visual exploration of the 150-sample dataset:

- Histograms, box plots, and violin plots per feature per species
- Full 4Г—4 pair plot with KDE diagonals
- Correlation heatmap (petal length vs petal width: **r = 0.96**)
- Andrews Curves and RadViz separability analysis
- SVM decision boundary preview

**Key finding:** Setosa is linearly separable from the other two species on every feature. Versicolor and Virginica overlap slightly, which is why non-linear or well-tuned classifiers are needed.

рҹ““ [`notebooks/iris_phase1_EDA.ipynb`](notebooks/iris_phase1_EDA.ipynb)

## рҹӨ– Phase 2 вҖ” Model Training & Comparison

Five classifiers trained, cross-validated, and compared head-to-head:

| Model | Test Accuracy | CV Mean | F1 Score |
|---|---|---|---|
| KNN | 93.3% | 96.7% | 0.933 |
| Logistic Regression | 93.3% | 95.3% | 0.933 |
| **SVM (tuned)** | **96.7%** | **97.3%** | **0.967** |
| Random Forest | 93.3% | 94.7% | 0.933 |
| XGBoost | 93.3% | 94.0% | 0.933 |

**Winner:** SVM with `C=10`, linear kernel, selected via `GridSearchCV` across 32 hyperparameter combinations and validated with 5-fold stratified cross-validation.

Feature importance (via Random Forest): **petal length + petal width = 86%** of total importance вҖ” sepal measurements barely matter.

## вҡҷпёҸ Phase 3 вҖ” Backend API

A FastAPI service that loads the trained model and exposes it over REST.

### Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Health check |
| `POST` | `/predict` | Predict species + confidence + probabilities + 3 nearest neighbors |
| `GET` | `/dataset` | Full dataset with PCA 3D coordinates (for frontend visualization) |
| `GET` | `/model-stats` | Live accuracy, F1, precision, recall, feature importance |
| `GET` | `/docs` | Auto-generated Swagger UI |

**рҹҢҗ Live API:** [iris-flower-classification-production-d9b1.up.railway.app/docs](https://iris-flower-classification-production-d9b1.up.railway.app/docs) вҖ” deployed on Railway

### Run it locally

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Visit `http://localhost:8000/docs` for the interactive Swagger UI.

## рҹҺЁ Phase 4 вҖ” Frontend

A single-file interactive web app (`frontend/iris_4d.html`) вҖ” no build step, no npm, just open it in a browser.

**Features:**
- Live sliders that call the API and update predictions in real time
- Rotating, zoomable 3D scatter plot of all 150 flowers (Three.js), built from PCA-reduced coordinates returned by the API
- A 4th "dimension" encoded as pulse/wave frequency, driven by petal width
- Species info card with real facts about whichever species is predicted
- Feature contribution breakdown вҖ” which measurement drove this specific prediction
- Session history strip вҖ” tracks your last 6 predictions
- Resilient design: if the 3D library fails to load, the rest of the app keeps working

**рҹҢҗ Live site:** [iris-flower-classification-production.netlify.app](https://iris-flower-classification-production.netlify.app) вҖ” deployed on Netlify

### Run it locally

1. Start the backend (see Phase 3 above)
2. Open `frontend/iris_4d.html` directly in your browser

> Note: the deployed version of `iris_4d.html` points to the live Railway API. To run fully locally, the `API` constant inside the file's `<script>` should point to `http://localhost:8000` instead.
